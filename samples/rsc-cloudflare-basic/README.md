# rsc-cloudflare-basic

Minimal consumer of `@yoshikouki/hono-rsc-renderer` using the official
`@cloudflare/vite-plugin` integration.

The `rsc` environment is the Worker and owns `ssr` as a child environment. The
SSR output stays under `dist/rsc`, so the generated `dist/rsc/wrangler.json`
describes a self-contained deploy bundle. The renderer continues to discover
its SSR entry through `import.meta.viteRsc.import()`.

The same Worker route serves HTML by default and Flight when the request sends
`RSC: 1` or `Accept: text/x-component`.

```sh
bun run dev
bun run build
bun run preview
bun run deploy:dry-run
```

`bun run test` verifies HTML and Flight responses in both dev and preview,
rebuilds the sample, and runs Wrangler's deploy dry run against the configuration
generated by the Cloudflare Vite plugin. No custom workerd runner or build lock
is required.
