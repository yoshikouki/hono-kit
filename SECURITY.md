# Security Policy

## Supported Versions

`hono-kit` is pre-release software. Until the first stable release, security
fixes target the `main` branch and the latest published package versions, if any.

## Reporting a Vulnerability

Please do not open a public issue with vulnerability details.

Use GitHub private vulnerability reporting for this repository when it is
available. If that option is not available, open a public issue that asks for a
private contact path without including exploit details, affected secrets,
private URLs, or proof-of-concept payloads.

Helpful reports include:

- Affected package and version or commit
- Impacted API, route, or generated output
- Reproduction steps
- Expected and observed behavior
- Any known workarounds

The maintainer will confirm receipt, triage the report, and coordinate a fix or
disclosure path based on severity and exploitability.
